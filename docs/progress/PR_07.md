# PR-07 - Epic 17 GrooveMap / PerformanceMap 2.0

## Fecha

2026-06-25

## Resumen

Se implemento `PerformanceMap` como capa comun de performance para material
rule-based y material importado desde modelos. La partitura mantiene starts y
duraciones cuantizadas para no romper validadores ni MusicXML, pero el export
MIDI aplica microtiming desde anotaciones de performance.

## Funcionalidad anadida

- `PerformanceMap`:
  - usa `GrooveMap` guardado en `metadata.song_plan.groove_map`;
  - conserva `swing_ratio`, `feel` y `meter`;
  - define rangos de velocity por rol;
  - define microtiming por rol y por material importado.
- `PerformanceMapper`:
  - aplica performance idempotente;
  - anota `performance_source`;
  - mantiene compatibilidad con `humanized_timing_ms`;
  - evita doble humanizacion si `performance_applied` ya existe;
  - preserva microtiming expresivo importado;
  - normaliza velocities de material IA.
- Fuentes soportadas:
  - `rule_based`;
  - `imported_model`;
  - `normalized_model`.
- `ProjectMerger`:
  - normaliza material importado despues del merge temporal;
  - la take pendiente ya contiene notas IA con `performance_source:
    normalized_model`.
- `TakeManager`:
  - aplica PerformanceMap de forma idempotente al aceptar take.
- Export MIDI:
  - lee `performance_microtiming_ms` o `humanized_timing_ms`;
  - convierte ms a ticks segun tempo;
  - acota el start dentro del compas;
  - no altera MusicXML ni duraciones notadas.

## Flujo validado

```text
rule_based generation
-> SongPlan/GrooveMap
-> PerformanceMap
-> notes annotated as performance_source=rule_based
-> MIDI export applies microtiming
```

```text
model artifact
-> import
-> temporary merge
-> PerformanceMap normalization
-> notes annotated as performance_source=normalized_model
-> validation/take pending/accept
```

## Notas de implementacion

- PerformanceMap no modifica `NoteEvent.start` ni `NoteEvent.duration`; solo
  modifica velocity y metadata de performance.
- El microtiming se materializa en MIDI, no en MusicXML.
- El material importado pasa de `imported_model` a `normalized_model` al quedar
  integrado en una take candidata.
- Si un artifact trae timing expresivo previo, se preserva y se registra
  `performance_timing_status: preserved_imported`.
