# PR-06 - Epic 16.9 Text2MIDI Sketch Backend opcional

## Fecha

2026-06-25

## Resumen

Se implemento Text2MIDI como backend experimental para bocetos simbolicos desde
prompt. El resultado no se mezcla con proyectos profesionales ni crea export
final: queda como `ArrangementProject` experimental en storage de sketches, con
artifact en cuarentena, clasificacion de roles, validacion y estado
`sketch_valid`, `sketch_uncertain` o `sketch_rejected`.

Endpoint nuevo:

```text
POST /v1/ai/text-to-midi-sketch
```

## Funcionalidad anadida

- `Text2MidiBackend`:
  - soporta solo `generate_full_sketch`;
  - mantiene dependencias pesadas opcionales y lazy;
  - no importa `text2midi`, `torch` ni `transformers` en import-time;
  - devuelve error controlado si Text2MIDI no esta instalado;
  - materializa salidas como `mido.MidiFile`, bytes, path o dict con path;
  - marca artifacts con `sketch_only: true`.
- `Text2MidiSketchImporter`:
  - lee MIDI multitrack o por canales;
  - convierte notas MIDI a `ArrangementProject`;
  - rellena barras con rests para preservar duraciones validas;
  - infiere tempo, compas y bar count;
  - clasifica roles por nombre de track, canal, programa GM y registro;
  - marca `sketch_uncertain` si no hay roles detectables o hay baja confianza;
  - rechaza sketches sin notas, MIDI corrupto o validacion con errores.
- API `ai_generation`:
  - anade `POST /v1/ai/text-to-midi-sketch`;
  - guarda artifact raw con `ArtifactStore`;
  - importa el sketch como proyecto experimental bajo `sketches/{sketch_id}`;
  - escribe `arrangement_project.json`, `validation_report.json`,
    `validation_report.html` y `sketch_metadata.json`;
  - nunca escribe en `projects/{id}` ni acepta el sketch como arreglo final.
- `MockSymbolicBackend`:
  - anade fixture `unlabeled_midi` para probar `sketch_uncertain`.

## Flujo validado

```text
prompt
-> backend.generate(generate_full_sketch)
-> ArtifactStore raw
-> import MIDI sketch
-> role classifier
-> ArrangementProject experimental
-> validate_project
-> ArtifactStore validated o rejected
```

## Notas de implementacion

- La config por defecto mantiene `text2midi` desactivado. Para usarlo realmente,
  hay que habilitarlo en `configs/ai_models.yaml` o via `AI_MODELS_CONFIG` e
  instalar Text2MIDI en el perfil opcional de modelos.
- El adapter real es flexible porque las APIs publicas de Text2MIDI pueden
  variar. Busca pipelines/clases comunes y metodos `generate_full_sketch`,
  `text_to_midi`, `generate_midi` o `generate`.
- El sketch no tiene `chord_grid` inferido; queda documentado como limitacion y
  requiere revision manual antes de convertirse en material profesional.
- El endpoint no crea takes pendientes de proyectos existentes. Eso es
  intencional: PR-06 solo importa bocetos experimentales.

## Riesgos pendientes

- Text2MIDI real puede requerir adaptacion fina segun el paquete instalado.
