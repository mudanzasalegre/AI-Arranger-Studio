# Progress - PR-16 Release Quality Gate

## Fecha

2026-06-29

## Resumen

Se implemento un gate de release para impedir exportaciones finales con material
invalido. El export profesional ahora valida, ademas de MIDI/MusicXML, que no
haya takes pendientes, que la take activa este aceptada, que las licencias de
modelo sean compatibles con el modo de export y que los manifests de dataset
declarados no tengan licencias bloqueantes.

## Funcionalidad anadida

- Nuevo modulo `arranger_core.release_gate`.
- Nuevo contrato:
  - `RELEASE_GATE_VERSION`;
  - `ReleaseExportMode`;
  - `validate_release_quality`.
- El gate valida:
  - errores bloqueantes de validacion del proyecto;
  - estado del export manifest;
  - presencia de `model_trace_json`, `takes_manifest_json` y `session_readme`;
  - active take aceptada;
  - takes pending bloqueantes;
  - rejected takes con metadata de rechazo/validacion;
  - accepted model takes con validation `pass` o `pass_with_warnings`;
  - `model_trace.json` sin backend/tarea de audio;
  - `commercial_use=forbidden` siempre bloquea;
  - `EXPORT_MODE=commercial` o `export_mode=commercial` exige
    `commercial_use=allowed`;
  - manifests de dataset declarados en `project.metadata` o
    `generation_spec.constraints`;
  - licencias de dataset desconocidas/bloqueadas;
  - uso comercial de datasets cuando el export es comercial.
- `export_project(...)` ejecuta el release gate y fusiona el resultado en
  `validation_report.json/html`.
- API acepta `export_mode` en:
  - `GenerateOptions`;
  - `ProjectExportRequest`.
- API devuelve `422` con el validation report cuando el release gate falla.

## Riesgos pendientes

- El gate de similitud/memorizacion profunda queda limitado a metadata
  disponible; la exportacion de datasets/tokenizacion de PR-17 podra aportar
  senales mas completas.
- El gate valida manifests de dataset declarados. Si un proyecto usa patrones sin
  registrar manifest en metadata/constraints, solo se puede auditar la
  trazabilidad presente en los eventos.
- El modo comercial bloquea modelos no permitidos, pero la revision legal humana
  sigue siendo necesaria para backends externos reales.
