# Progress - PR-14 App Workflow 2.0

## Fecha

2026-06-29

## Resumen

Se implemento la capa de workflow de aplicacion para IA controlada. La UI ahora
incluye una vista `AI workflow` con AI Plan, AI Infill, Generate Track, panel de
takes, diff/validacion, accept/reject/restore y workspace de sketches. La API
expone diff y preview de archivos por take sin modificar el proyecto activo.

## Funcionalidad anadida

- Nueva vista frontend `AI workflow`.
- Panel `AI Plan` conectado a `POST /v1/projects/{id}/ai/plan`.
- Panel `AI Infill` conectado a `POST /v1/projects/{id}/ai/infill`.
- Accion `Generate Track` que usa todo el rango de compases del proyecto.
- Panel de takes con seleccion, diff, accept, reject y restore de takes
  aceptadas no activas.
- Preview MIDI y JSON de proyecto para takes pendientes o aceptadas.
- Workspace de sketch conectado a `POST /v1/ai/text-to-midi-sketch`.
- Endpoint `GET /v1/projects/{project_id}/takes/{take_id}/diff`.
- Endpoint `GET /v1/projects/{project_id}/takes/{take_id}/file`.
- Diff de take por pista y compas:
  - pistas anadidas, eliminadas, modificadas o sin cambios;
  - compases modificados;
  - conteo de notas activo/candidato;
  - validacion asociada a la take.
- Preview MIDI generado desde el snapshot de la take, dentro de la carpeta de
  cuarentena de esa take, sin aceptar automaticamente el resultado.
