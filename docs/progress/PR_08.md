# PR-08 - Epic 18 Melody Engine 2.0

## Fecha

2026-06-25

## Resumen

Se implemento `MelodyEngine` como capa dedicada para generacion melodica
profesional. El motor conserva el modo rule-based por defecto, anade modo
retrieval desde `melodic_motifs` y prepara modo `ai_infill` mediante backend
inyectable con validacion y fallback seguro.

## Funcionalidad anadida

- `MelodyEngine`:
  - soporta `rule_based`, `retrieval` y `ai_infill`;
  - usa `LeadSheetGenerator` como base rule-based;
  - adapta patrones `melodic_motifs` de dataset cuando hay retrieval;
  - valida rango, respiracion, duracion y soporte armonico;
  - vuelve a rule-based si retrieval o IA fallan;
  - no acopla el core a `model_backends`.
- `MotifLedger`:
  - registra frases, compases, fuente, modo, motivo, variacion, cadencia,
    ritmo y contorno;
  - registra material aceptado desde `ai_infill`;
  - queda guardado en metadata de la pista melodica.
- Integracion con `MelodyGenerator`:
  - sustituye el uso directo de `LeadSheetGenerator`;
  - conserva compatibilidad con la salida anterior;
  - mantiene `MotifLedger` incluso cuando la melodia se fusiona dentro del
    piano en trios.
- Retrieval melodico:
  - lee `melodic_motifs` desde `context.learned_patterns`;
  - transpone motivos al acorde activo;
  - no copia literal el material del dataset;
  - rellena silencios para mantener compases validos.
- AI infill:
  - define protocolo `MelodyAiInfillBackend`;
  - permite generar solo compases/frases objetivo;
  - valida antes de aceptar;
  - fallback rule-based si no hay backend, si falla el backend o si no valida.

## Flujo validado

```text
SongPlan / PhrasePlan
-> MelodyEngine
-> rule_based | retrieval | ai_infill
-> melody validation
-> MotifLedger
-> PerformanceMap
-> export MIDI/MusicXML
```

## Notas de implementacion

- `melody_mode` o `melody_engine_mode` en `GenerationSpec.constraints` permite
  forzar `rule_based`, `retrieval` o `ai_infill`.
- Si existe `pattern_index_path` o `pattern_index` con `melodic_motifs`, el
  modo puede pasar a retrieval salvo que `melody_retrieval` sea `false`.
- `ai_infill` queda preparado como interfaz de motor; los endpoints IA
  controlados siguen usando la cuarentena/takes de PR-03 a PR-05.
- El motor no implementa PR-09, PR-10 ni PR-11.

## Riesgos pendientes
