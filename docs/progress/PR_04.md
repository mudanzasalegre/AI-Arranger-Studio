# PR-04 - Epic 16.7 LLM Planner JSON

## Fecha

2026-06-25

## Resumen

Se implemento la capa de planificacion LLM simbolica segura. El planner acepta
JSON validado por Pydantic para producir o parchear estructura musical, pero no
puede producir notas finales, MIDI, MusicXML, audio, lyrics ni exports.

El endpoint nuevo es:

```text
POST /v1/projects/{id}/ai/plan
```

El endpoint guarda una version de plan y actualiza solo metadatos de
planificacion del proyecto. Las pistas del arreglo activo no se modifican.

## Funcionalidad anadida

- Schemas Pydantic:
  - `LlmSongPlanPatch`
  - `LlmSectionPatch`
  - `RoleIntent`
  - `GenerationStrategy`
  - `LlmPlannerResult`
  - `PlanAttempt`
- `PlanValidator`:
  - rechaza JSON invalido;
  - rechaza campos de salida prohibidos como MIDI, MusicXML, audio, exports,
    lyrics o note events;
  - valida estilos, formas, ensembles e instrumentos contra catalogos;
  - valida tempo, key, meter y rangos de secciones;
  - rechaza secciones solapadas;
  - rechaza objetivos bloqueados por `locked_tracks` o `locked_sections`.
- `LlmPlanner`:
  - acepta un proveedor JSON inyectable para tests o integraciones futuras;
  - reintenta una vez si el JSON falla;
  - pasa el error concreto al segundo intento;
  - si vuelve a fallar, cae a `PromptCompiler` + `SongPlanner`;
  - convierte patches validos a `SongPlan`, `SectionPlan`, `PhrasePlan` y
    `GrooveMap`;
  - no genera `NoteEvent`, MIDI, MusicXML ni audio.
- API:
  - `POST /v1/projects/{id}/ai/plan`;
  - guarda `plan_versions/{plan_version}.json`;
  - guarda `song_plan.json`;
  - actualiza `metadata.song_plan`, `metadata.active_plan_version`,
    `metadata.plan_versions` y `metadata.ai_planner`;
  - verifica que las pistas no cambien durante la planificacion.

## Notas de implementacion

- No se integra ningun proveedor LLM real en este PR; queda preparado mediante
  `PlannerJsonProvider`.
- Sin proveedor configurado, el endpoint usa fallback rule-based y devuelve
  `planner=fallback_rule_based`.
- El resultado se versiona, pero no crea takes. La cuarentena/takes queda para
  material MIDI de PR-05/PR-06.
- El planner modifica solo estructura de plan. No toca tracks, notas, exports
  ni archivos MIDI.
