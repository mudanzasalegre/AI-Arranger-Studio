# Progress - PR-13 Retrieval Model

## Fecha

2026-06-28

## Resumen

Se implemento un modelo comun de retrieval para patrones simbolicos. Los
generadores ya no eligen simplemente el primer patron compatible: consultan un
`PatternRetriever`, rankean por similitud musical/contextual y pasan por un
`PatternAdapter` que transforma el payload antes de usarlo para evitar copia
literal.

## Funcionalidad anadida

- Contrato de retrieval:
  - `RetrievalQuery`
  - `RetrievalCandidate`
  - `PatternRetriever`
  - `PatternAdapter`
  - `retrieve_pattern`
  - `retrieve_patterns`
  - `retrieval_trace`
- Ranking por:
  - categoria;
  - rol y alias de rol;
  - estilo y familia estilistica;
  - instrumento;
  - densidad;
  - metrica;
  - contexto armonico;
  - calidad/peso;
  - confianza de role classifier;
  - sensibilidad/licencia.
- Adaptacion de patrones:
  - `walking_bass_cells`: rotacion/ornamentacion de intervalos;
  - `melodic_motifs`: variacion de grados relativos;
  - `piano_voicings`: variacion controlada de intervalos de voicing;
  - `drum_grooves`: desplazamiento de onsets;
  - `horn_responses`: variacion de notas relativas;
  - `progressions`: rotacion de progresion.
- Trazabilidad:
  - `transformations_applied`;
  - `similarity_to_source`;
  - `source_pattern_id`;
  - `source_file_hash`;
  - query usada por el retriever.
- Integracion en generadores:
  - `BassEngine`;
  - `DrumsEngine`;
  - `MelodyEngine`;
  - `PianoCompingEngine`.

## Riesgos pendientes

- El ranking es heuristico y determinista; aun no usa embeddings ni un indice
  vectorial. Eso es suficiente para PR-13 y deja una base clara para evolucionar.
- La adaptacion reduce copia literal a nivel de payload, pero el gate fuerte de
  similitud/memorizacion sigue correspondiendo a PR-16/PR-17.
- `pad`, `solo` y retrieval por progresiones quedan soportados en contrato, pero
  todavia no estan conectados a motores de generacion dedicados.
- La UI/API de takes/diff para retrieval queda para PR-14.
