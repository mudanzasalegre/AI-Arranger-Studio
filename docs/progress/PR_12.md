# Progress - PR-12 Dataset Profiler / Role Classifier

## Fecha

2026-06-28

## Resumen

Se implemento una capa de profiling y clasificacion de roles para datasets MIDI
y MusicXML. `import_dataset` ahora deja cada libreria importada con perfil por
archivo, manifest de roles, features musicales, metadatos de licencia comercial
y fingerprints anti-memorizacion para que el material quede listo para retrieval
y futuros modelos propios.

## Funcionalidad anadida

- Modelos nuevos:
  - `RoleClassification`
  - `TrackProfile`
  - `DatasetFileProfile`
  - `DatasetProfileReport`
- Roles clasificados:
  - `melody`
  - `bass`
  - `drums`
  - `comping`
  - `horns`
  - `pad`
  - `solo`
  - `harmony`

- Features por pista:
  - registro, pitch span, densidad, notas por compas;
  - ratio de polifonia/chord onsets;
  - ratio de canal de bateria;
  - duraciones largas/cortas;
  - repeticion ritmica y saltos medios.
- Features por archivo:
  - formato, tracks/parts, note count, chord symbols;
  - role coverage;
  - `contains_melody`, `contains_chords`, `contains_arrangement`.
- Licencia y uso:

  - `pattern_sensitivity`.
- Fingerprints:
  - fingerprint por track;
  - fingerprint por archivo;

- Salidas nuevas de import:

## Riesgos pendientes

- La clasificacion es heuristica y explicable, no un clasificador entrenado.
  Funciona para alimentar PR-13, pero debera calibrarse con corpus reales.
- `pad` y `solo` quedan perfilados, pero todavia no tienen categorias de pattern
  extraction propias; PR-13/PR-17 deben decidir segmentos y tokenizacion.
- La similitud anti-memorizacion queda trazada mediante fingerprints; el gate de
  bloqueo fuerte corresponde a PR-16/PR-17.
- `commercial_training` deriva licencias permisivas conocidas, pero licencias
  ambiguas siguen en `review_required`.
