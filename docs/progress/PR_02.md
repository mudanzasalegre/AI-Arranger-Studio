# PR-02 - Epic 16.5 AI Model Backend Contract

## Fecha

2026-06-25

## Resumen

Se implemento la capa comun `packages/model_backends` para integrar modelos
simbolicos sin acoplar el core ni la API a MIDI-GPT, Text2MIDI u otros backends
con dependencias pesadas.

La API ya expone:

```text
GET /v1/ai/models
```

El backend `mock_symbolic` queda disponible por defecto. `midigpt` y
`text2midi` quedan configurados como desactivados y no se importan en import-time.

## Funcionalidad anadida

- Contrato Pydantic:
  - `ModelCapabilities`
  - `ModelGenerationRequest`
  - `ModelGenerationResult`
  - `ModelArtifact`
  - `MusicModelBackend`
- Registry:
  - alta/listado de backends;
  - estados `available`, `unavailable`, `disabled`;
  - error controlado al pedir backends desactivados o no disponibles.
- Config:
  - carga de `configs/ai_models.yaml`;
  - soporte `AI_MODELS_CONFIG`;
  - registro condicional de backends desactivados/no disponibles.
- Mock backend:
  - fixture MIDI valido;
  - fixture MIDI invalido;
  - artifact JSON valido;
  - error simulado.
- Backends opcionales:
  - `MidiGptBackend` sin import-time de `midigpt`;
  - `Text2MidiBackend` sin import-time de Text2MIDI;
  - errores controlados si faltan dependencias.
- API:
  - `GET /v1/ai/models` lista mock disponible y opcionales desactivados.
