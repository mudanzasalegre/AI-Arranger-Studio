# PR-05 - Epic 16.8 MIDI-GPT Backend opcional

## Fecha

2026-06-25

## Resumen

Se implemento el flujo de infill simbolico controlado para MIDI-GPT y mock
backends. El proyecto activo no se modifica automaticamente: el resultado del
modelo se guarda como artifact raw, se importa, se fusiona en una copia temporal,
se valida y, si pasa, queda como take pendiente.

Endpoint nuevo:

```text
POST /v1/projects/{id}/ai/infill
```

## Funcionalidad anadida

- `MidiGptBackend`:
  - mantiene carga lazy de `midigpt`;
  - no importa `midigpt`, `torch` ni `transformers` en import-time;
  - reporta error controlado si falta la dependencia;
  - soporta `infill_bars`, `generate_track`, `continue_section` y
    `generate_variation`;
  - carga `midigpt.inference.engine.InferenceEngine` solo en ejecucion;
  - materializa salidas como `mido.MidiFile`, bytes, path o dict con path;
  - registra `model_name`, `track_id`, `bars`, `instruction`, `temperature` y
    `seed` en metadata.
- `MockSymbolicBackend`:
  - conserva fixture MIDI valido;
  - conserva fixture MIDI invalido;
  - anade fixture `empty_midi` para forzar fallo de validacion musical.
- API `ai_generation`:
  - exporta contexto MIDI temporal desde `ArrangementProject`;
  - llama al backend seleccionado (`midigpt` o `mock_symbolic`);
  - fuerza `artifact_raw_dir` al root de cuarentena activo para evitar fugas a
    `outputs/model_artifacts` durante tests;
  - guarda artifact raw en `ArtifactStore`;
  - importa con `ArtifactImporter`;
  - fusiona solo `track_id` y `bars` solicitados con `ProjectMerger`;
  - valida con `ValidationGate`;
  - crea take pendiente con `TakeManager`;
  - rechaza artifacts invalidos o candidatos que fallen validacion.
- `TakeManager`:
  - acepta metadata opcional para guardar `model_trace` en la take.

## Flujo validado

```text
ArrangementProject
-> context MIDI temporal
-> backend.generate(ModelGenerationRequest)
-> ArtifactStore raw
-> ArtifactImporter
-> ProjectMerger solo target track/bars
-> ValidationGate
-> TakeManager pending
```

## Notas de implementacion

- La config por defecto mantiene `midigpt` desactivado. Para usarlo realmente,
  hay que habilitarlo en `configs/ai_models.yaml` o via `AI_MODELS_CONFIG` e
  instalar `pip install "midigpt[inference]"`.
- Si se usa `AI_ARRANGER_API_STORAGE`, los artifacts del backend se escriben en
  ese storage, no en la cuarentena global.
- El endpoint no acepta el resultado como proyecto activo. Siempre crea take
  pendiente.
- El flujo no sustituye Harmony Engine, validadores ni exportadores.

## Riesgos pendientes

- La llamada real a MIDI-GPT queda adaptada de forma flexible porque la API
  exacta del paquete puede variar.
- No hay retry policy completa de temperatura/reparacion; PR-05 deja el camino
  seguro base y los PRs posteriores pueden sofisticarlo.
- Los validadores musicales especificos por instrumento todavia son estructurales
  y se reforzaran en los PRs de motores 2.0.
