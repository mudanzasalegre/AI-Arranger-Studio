## Funcionalidad anadida

- `model_trace.json` en cada export:
  - `project_id`;
  - `active_take_id`;
  - `trace_scope`;
  - `model_artifacts` de la take modelo aceptada activa.
- Cada entrada de `model_artifacts` registra:
  - backend;
  - task;
  - prompt/instruction;
  - track_id;
  - bars;
  - seed;
  - take_id;
  - validation_result;
  - commercial_use.
- `takes_manifest.json` exportado en raiz con politica `accepted_only`.
- El manifest de takes exportado elimina:
  - takes pending;
  - takes rejected;
  - `project_snapshot_path`;
  - rutas internas de validation/context/artifacts.
- `session_readme.md` con una guia breve de uso DAW/MuseScore.
- `validate_export_package` exige ahora:
  - `model_trace_json`;
  - `takes_manifest_json`;
  - `session_readme`.
- `GET /v1/projects/{project_id}/zip` empaqueta solo los archivos listados en
  `export_manifest.json`.
- Aliases API para abrir:
  - `kind=model_trace`;
  - `kind=takes_manifest`;
  - `kind=readme`.
- El smoke de demo/packaging exige los nuevos archivos del paquete final.
